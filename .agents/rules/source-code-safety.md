# Безопасность и правила записи

**Single source of truth по зонам записи и правилу апрува.**

## Правило апрува (главное)

ИИ **не реализует и не удаляет ничего без явного апрува пользователя**. Порядок всегда: research → план/раскладка → апрув → код. Это относится к любому коду, тестам, скиллам, массовым правкам.

## Секреты

- Ключ Nessie и любые токены — **только в `.env`** (gitignored). Никогда в код, `config.properties`, `.agents/` или git.
- Перед любым коммитом убедиться, что `.env` не в индексе (`git status`). Правило `*.env` в `.gitignore` это ловит.

## Зоны записи

| Зона | Правило |
|---|---|
| `src/**`, `pom.xml`, `.agents/**`, `CLAUDE.md` | Правка только **после апрува** конкретного изменения. |
| `PROGRESS.md` | Обновляется после каждого закрытого шага (это регламент проекта). |
| `.env` | Только локально, руками пользователя. Не коммитить. |

## Git

- Коммиты выполняет пользователь на своей машине (у песочника ИИ бывает залипание блокировки `.git`). ИИ готовит изменения и даёт готовую команду коммита.
- ИИ не пушит от имени пользователя.

## Границы

- Монорепо `sourcing-e2e` и его `.agents/` — **чужой проект-донор**. Использовался как образец техник, но не является частью этого репозитория. Не смешивать, не редактировать, не считать своей зоной.
