# Конвенции кода

Как писать код в этом проекте. Обоснование слоёв — `architecture.md`.

## Контекст над кодом (обязательно)

Каждый класс и значимый метод несёт **структурированный Javadoc-контекст** — техника монорепо sourcing-e2e (`@tool`/`@component`/`@behavior`), адаптированная под Java. Это не «тупые комменты», а теги со смыслом и связями.

Теги:
- **Над классом:** `@layer <config|routes|specs|api|base|tests>`, `@domain <домен>` (если применимо), `@context <1 строка смысла>`, `@see .agents/rules/architecture.md`.
- **Над методом клиента (`api/`):** `@endpoint <VERB /path>`, `@behavior <что делает>`, `@returns` (raw Response, без ассертов).
- **Над методом теста:** `@scenario <что проверяем>`.
- **Над методом Config/утилит:** `@behavior <1 строка>`.
- **Над роут-константой:** `@endpoint <что за путь>`.

Правила:
- **Inline-«мусор» вне закона** — контекст живёт над сигнатурой, не внутри тела метода.
- Здесь — только локальный смысл элемента; глубокое «почему/архитектура» — в `.agents/rules/`.
- Новый архитектурный нюанс — сначала в `.agents/rules/`, потом код.

## Язык

- **Сообщения исключений и логов — English** (единый язык логов, проще грепать/гуглить стектрейсы).
- **Javadoc и редкие пояснения — русский** (язык команды).

## Слои — где что живёт

- **Тест** (`tests/`): только Arrange–Act–Assert. Никакого HTTP, URL, ключей. Обращается к доменному клиенту + `Specs`.
- **API-клиент** (`api/`): выполняет запрос, отдаёт `Response`. **Никаких ассертов внутри.**
- **Пути**: только через `Routes`-константы, не строками в коде.
- **Общие настройки запроса/ответа**: только через `Specs` (не копипастить в тестах).
- **Секреты/URL**: только через `Config`.

## Именование

- Методы клиента: по действию — `getAll()`, `getById(...)`, `create(...)`.
- Тесты: `методОжидаемыйРезультат` — например `getAccounts_returns200`. `@DisplayName` — человекочитаемо (можно на русском, это отчёт).
- Тест-класс: `<Domain>SmokeTest` / `<Domain>Tests`.

## Java-стиль

- **DTO — `record`** (иммутабельность), (де)сериализация Jackson — с Фазы 3.
- Утилитные классы (`Config`, `Routes`, `Specs`): `final class` + приватный конструктор.
- Fail-fast на конфиге: нет обязательного значения — бросаем `IllegalStateException` с внятным English-сообщением.
- `try-with-resources` для любых потоков/ресурсов.

## Зависимости

- Всё `scope=test` (см. `architecture.md`). Новые зависимости — по фазам из `PROGRESS.md`, не пачкой.

## Секреты

- Ключ Nessie — только в `.env`. Никогда в код, `config.properties` или git. Детали зон — `source-code-safety.md`.
