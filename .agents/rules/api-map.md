# Карта тестируемого API (Nessie)

Лёгкий «каталог» внешнего API — чтобы ИИ ориентировался без чтения чужой документации целиком. **Описательный: заполняется по мере добавления клиентов, ничего не выдумываем.** Смысл сущностей — `glossary.md`.

## Базовое

- **Base URL:** `http://api.nessieisreal.com` (именно `http`, у sandbox нет SSL). Хранится в `config.properties` (`base.url`).
- **Авторизация:** ключ в каждом запросе как query-параметр `?key=...`. Ключ — в `.env` (`NESSIE_API_KEY`).
- **Два ключа с разными правами:** *Enterprise* (роль сотрудника, только `GET`) и *Customer* (роль клиента, все операции над своими сущностями). Для CRUD нужен Customer key.
- **Формат:** JSON (`application/json`).

## Домены

`customers`, `accounts`, `transfers`, `deposits`, `withdrawals`, `bills`, `purchases`, `merchants` (значения — `glossary.md`).

## Подтверждённые эндпоинты

Заполняем по мере покрытия. Проверенное на практике:

| Метод | Путь | Статус | Клиент | Заметки |
|---|---|---|---|---|
| GET | `/accounts` | 200 | `AccountsApi.getAll()` | Возвращает массив аккаунтов ключа; пустой = `[]`. |

Остальные эндпоинты добавляем сюда при создании соответствующих клиентов (рецепт — `../skills/add-endpoint/SKILL.md`). Точные пути/тела брать из документации Nessie (`http://api.nessieisreal.com/documentation`) или OpenAPI, не угадывать.
