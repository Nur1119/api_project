# .agents/ — точка входа для ИИ

**Открой этот файл первым.** Он — карта контекста проекта и маршрутизатор: говорит, какой файл открыть под конкретную задачу, чтобы не грепать весь репозиторий и не жечь токены.

Принцип: **single source of truth**. Каждый факт живёт в одном файле; остальные ссылаются, не дублируют. Если факт устарел — правится в одном месте.

## Что за проект (одной строкой)

Автотесты REST API банковского sandbox Capital One Nessie на Java. Полный роадмап и живой статус — в `../PROGRESS.md` (корень проекта).

## Маршрут по задачам

| Задача | Открой |
|---|---|
| Понять раскладку проекта (пути + смысл) | `rules/project-map.md` |
| Понять архитектуру: слои, поток данных, принятые решения | `rules/architecture.md` |
| Как писать код в этом проекте (конвенции) | `rules/conventions.md` |
| Куда можно/нельзя писать, правило апрува, секреты | `rules/source-code-safety.md` |
| Термины домена Nessie и стека | `rules/glossary.md` |
| Поверхность тестируемого API (домены, эндпоинты, авторизация) | `rules/api-map.md` |
| Добавить новый эндпоинт/домен (рецепт по слоям) | `skills/add-endpoint/SKILL.md` |

## Порядок для новой сессии (cold start)

1. `rules/source-code-safety.md` — что можно трогать и правило апрува.
2. `rules/project-map.md` + `rules/architecture.md` — где что и почему так.
3. `../PROGRESS.md` — на каком шаге стоим.
4. Только потом — конкретная задача (через таблицу выше).

## Правила ведения этого слоя

- Документы **описательны**: фиксируют то, что реально есть в коде и что уже решено. Ничего не выдумывают.
- Новый факт — в профильный файл, а не в комментарий кода. Комментарии в коде минимальны (см. `rules/conventions.md`).
- Меняется архитектура/решение — сначала правим `rules/`, потом код.
