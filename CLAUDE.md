## Что за проект
Учебно-практический проект: автотесты REST API банковского sandbox (Capital One Nessie) на Java. Цель — освежить и закрепить стек и подготовиться к собесам по API-тестированию в финтехе. Карты кода/API/архитектуры — в `.agents/` (вход: `.agents/README.md`).

## Кто работает
Nur — QA-автоматизатор, 2.5 года на TypeScript + Playwright (UI-автотесты). Java не трогал ~2.5 года, освежает. **Не новичок:** программирование, ООП, тестирование знает хорошо. Нужна Java-специфика и стек, без объяснений базовых вещей.

## КАК работать (важнее всего)
- **Апрув до реализации.** Ничего не пишу и не удаляю без явного «да» пользователя. Порядок: research → план/раскладка → апрув → код. (Канон: `.agents/rules/source-code-safety.md`.)
- **Учи по ходу дела.** Это обучение. Каждый значимый шаг объясняй: что, зачем, как спрашивают на собесе. Не вываливай код без разбора.
- **Без воды.** ADHD-friendly: плотно, по делу, без вступлений и филлера.
- **Язык.** Объяснения и Javadoc — русский; сообщения исключений/логов — English. (Канон: `.agents/rules/conventions.md`.)
- **Контекст, а не комменты.** Знание «почему» — в `.agents/rules/`, не в комментариях кода. Комментарии минимальны.
- **Дисциплина фаз.** Строго по `PROGRESS.md`, по порядку. Не перескакивать. Не обсуждать будущие фазы, пока текущая не закрыта.
- **Веди прогресс.** После каждого шага обновляй чекбоксы, «Текущий статус» и «Журнал» в `PROGRESS.md`. Коммит выполняет пользователь.
- **Секреты не коммить.** Ключ Nessie — только в `.env`, никогда в git.

## Источник правды
`CLAUDE.md` (loader) + `.agents/` (карты контекста) + `PROGRESS.md` (статус). Любая сессия восстанавливает контекст отсюда — держи их в порядке.
