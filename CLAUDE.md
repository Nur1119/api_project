# CLAUDE.md — контекст и правила для Claude Code

Этот файл Claude Code читает автоматически в начале каждой сессии. Здесь — что за проект, кто работает и КАК работать.

## Что за проект
Учебно-практический проект: автоматизированное тестирование REST API банковского приложения (Capital One Nessie sandbox) на Java. Цель — не просто написать тесты, а **освежить и закрепить стек** и подготовиться к собеседованиям по API-тестированию в финтехе.

## Кто работает
Nur — QA-автоматизатор, 2.5 года на TypeScript + Playwright (UI-автотесты). Java не трогал ~2.5 года, сейчас освежает. **Не новичок:** программирование, ООП, тестирование знает хорошо. Нужно вспоминать именно Java-специфику и стек ниже, без объяснений базовых вещей.

## Стек
Java 25 · Maven · JUnit 5 · REST Assured · Jackson · (Lombok) · WireMock · Docker · GitHub Actions (CI).
Зависимости добавляются по фазам, не всё сразу.

## Тестируемое API
- Базовый URL: `http://api.nessieisreal.com`
- Ключ передаётся как query-параметр `?key=...`
- Ключ хранится в env-переменной, **в git не коммитится** (настрой `.gitignore`).
- Домены: customers, accounts, transfers, deposits, withdrawals, bills, purchases, merchants.

## КАК работать (это важнее всего)
- **Учи по ходу дела.** Это обучение, а не молчаливая генерация кода. Каждый значимый шаг объясняй: что делаешь, зачем, и как это спрашивают на собесе. Не вываливай готовый код без разбора.
- **Без воды.** Nur — ADHD-friendly стиль: только полезное, плотно, по делу. Никаких длинных вступлений, повторов, филлера. Лишний текст = потраченное время.
- **Объяснения — на русском.** Технические термины — на английском (Java, REST Assured и т.д.).
- **Дисциплина фаз.** Иди строго по фазам из `PROGRESS.md`, по порядку. Не перескакивай через незавершённые шаги. **Не обсуждай будущие фазы, пока текущая не закрыта.**
- **Веди прогресс.** После каждого выполненного шага обновляй чекбоксы, «Текущий статус» и «Журнал» в `PROGRESS.md`, затем коммить осмысленным сообщением.
- **Секреты не коммить.** Ключ Nessie и любые токены — только в env, никогда в git.

## Статус и план
Полный роадмап, фазы и живой прогресс — в `PROGRESS.md`. **Всегда читай его в начале работы** и держи актуальным. Сейчас: Фаза 1 (окружение + каркас).

## Преемственность
`CLAUDE.md` + `PROGRESS.md` — источник правды. Любая сессия (эта, новая или другой ИИ) восстанавливает контекст отсюда. Поэтому держи оба файла в порядке.
